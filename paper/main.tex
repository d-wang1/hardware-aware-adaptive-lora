\begin{abstract}
Parameter-efficient fine-tuning methods like LoRA allow the adaptation
of large language models under tighter hardware and memory limitations.
However, standard LoRA allocates the same rank uniformly across all
adapted layers. This project studies whether a variable rank allocation
approach could be more effectively used under fixed hardware
constraints. Specifically, I experiment with a lightweight adaptive
LoRA method that, after a brief warmup phase, reallocates a fixed total
rank budget across layers a single time using a score
$s_i = g_i / c_i^{\alpha}$ that combines the gradient norms of LoRA
parameters with the per-rank parameter cost of each module. As opposed
to more complex adaptive budget allocation methods like those proposed
in papers like AdaLoRA, this approach is designed to be easy to
implement and operate explicitly under a fixed single-GPU budget. I
compare the proposed method against the standard uniform-rank LoRA,
AdaLoRA, and a gradient-only ablation ($\alpha{=}0$) on the SST-2
sentiment classification task by both statistical and hardware metrics.
Statistical metrics used include validation loss, validation accuracy,
and time to a $0.90$ accuracy target, and hardware evaluation includes
peak GPU usage, training throughput, scheduler overhead, and
wall-clock time. The goal of this project is to determine if a
low-overhead adaptive allocation rule can improve the quality and
efficiency of LoRA under low-resource constraints. The proposed method,
evaluated at $\alpha{=}1$, reaches the lowest trainable-parameter count
of any method considered ($1{,}081{,}346$ vs.\ AdaLoRA's
$1{,}366{,}562$), comparable accuracy with the lowest seed-to-seed
variance ($\pm 0.001$ over three seeds), and roughly a $19\times$
reduction in scheduler overhead relative to AdaLoRA. An $\alpha$-sweep
further shows the share of rank assigned to attention modules climbing
from $39.1\%$ to $56.1\%$ as $\alpha$ increases from $0.0$ to $1.0$,
suggesting that the hardware-cost term meaningfully reshapes the
allocation. The picture against uniform-rank LoRA is more mixed, with
peak memory and time-to-target favoring the uniform baseline on this
task.
\end{abstract}

\section{Introduction}
\label{sec:intro}

The growing scale of transformer-based language models has motivated
active research on parameter-efficient fine-tuning methods, of which
Low-Rank Adaptation (LoRA) \cite{hu2022lora} is one of the most widely
adopted. LoRA freezes the pretrained weights of the model and inserts
small trainable adapters of the form $\Delta W = BA$ into selected
linear layers, where $A \in \mathbb{R}^{r \times d}$ and
$B \in \mathbb{R}^{k \times r}$ are factors of rank $r$. By only
updating these adapters during fine-tuning, LoRA drastically reduces
the number of trainable parameters and the associated GPU memory
footprint, which makes it especially appealing in low-resource
settings. However, once the total rank budget $B = \sum_i r_i$ has
been fixed, there is still an additional design decision to be made:
how the budget should be distributed across the modules being adapted.
The standard formulation of LoRA addresses this by simply assigning
the same rank to every layer. More sophisticated methods such as
AdaLoRA \cite{zhang2023adalora} instead reallocate rank dynamically
during training by pruning the SVD components that contribute the
least to the loss. In both cases, rank is treated as if it were
fungible across the model, meaning that a single unit of rank added
at one module costs the same as a single unit added at another.

In practice, this treatment is an oversimplification. On
DistilBERT \cite{sanh2019distilbert}, adding one unit of rank to a
$768 \to 3072$ feed-forward projection introduces $2.5\times$ as many
trainable parameters as adding the same unit of rank to a
$768 \to 768$ attention projection. When the underlying constraint is
parameter count, GPU memory, or some closely related compute cost,
charging both modules the same ``rank price'' is therefore a clear
simplification. It is reasonable to ask whether a rank allocator that
explicitly takes this cost asymmetry into account could produce more
efficient configurations under fixed hardware constraints.

This paper studies a small but explicit modification to the LoRA rank
allocation rule, in which the gradient-based importance signal at
each module is scaled by the inverse of the per-rank parameter cost of
that module. For each LoRA target module $i$,
\begin{equation}
s_i \;=\; \frac{g_i}{c_i^{\alpha}},
\label{eq:rule}
\end{equation}
where $g_i$ is an exponential moving average of the LoRA gradient
norms collected during a brief warmup phase,
$c_i = d_i^{\text{in}} + d_i^{\text{out}}$ is the per-rank parameter
cost at module $i$, and $\alpha \in [0, 1]$ controls how strongly the
cost term is weighted. Ranks are then assigned across modules in
proportion to $s_i$ while the total budget is preserved. When
$\alpha = 0$, the rule collapses to a purely gradient-based allocator,
which serves as the ablation baseline; when $\alpha = 1$, the score
becomes the headline hardware-aware variant studied throughout this
paper.

I evaluate the resulting allocator on the SST-2 sentiment
classification task \cite{socher2013sst, wang2018glue} using
DistilBERT, comparing against Uniform LoRA,
AdaLoRA \cite{zhang2023adalora} (via the PEFT
implementation~\cite{mangrulkar2022peft}), and the $\alpha = 0$
gradient-only ablation. The contributions of this project are
threefold:
\begin{itemize}
\item the allocation rule in Eq.~\ref{eq:rule}, along with a
      budget-preserving discrete allocator that distributes integer
      ranks under fixed minimum and maximum constraints
      (\S\ref{sec:method});
\item a systems-level evaluation that reports peak GPU memory,
      training throughput, total wall-clock time, time-to-target,
      and scheduler overhead, with scheduler overhead surfacing as
      the clearest systems-level finding of this evaluation
      (\S\ref{sec:results-systems});
\item an $\alpha$-sweep that isolates the cost term and demonstrates
      how it monotonically reshapes the share of rank assigned to
      attention versus feed-forward modules, while final accuracy
      and the upstream gradient signal remain essentially unchanged
      (\S\ref{sec:results-alpha}).
\end{itemize}
This paper does not claim to universally outperform AdaLoRA in final
accuracy. Rather, the goal is to characterize the practical efficiency
tradeoffs that result from introducing a hardware-cost term into an
otherwise simple allocator, and to identify the conditions under which
the proposed rule does or does not deliver useful systems-level
improvements.

\section{Related Work}
\label{sec:related}

\textbf{LoRA} \cite{hu2022lora} introduced low-rank adapter
fine-tuning, in which every adapted layer receives the same fixed
rank. \textbf{AdaLoRA} \cite{zhang2023adalora} extends LoRA by
adaptively budgeting rank across modules using an SVD-based importance
score, and prunes unimportant components over the course of training.
The method proposed in this paper differs from AdaLoRA in two main
ways: the importance signal is a Frobenius-norm gradient EMA rather
than an SVD-based criterion, and that signal is explicitly divided by
the per-rank parameter cost of each module. \textbf{LoRA+}
\cite{hayou2024loraplus} addresses a different aspect of LoRA training,
namely the use of separate learning rates for the $A$ and $B$ factors,
and is essentially orthogonal to the question of budget allocation.
\textbf{QLoRA} \cite{dettmers2023qlora} composes LoRA with 4-bit
quantization of the base model, which is again orthogonal to how rank
is distributed across layers. \textbf{GaLore} \cite{zhao2024galore}
instead projects the optimizer state itself onto a low-rank subspace,
addressing the memory cost of full-parameter training rather than the
rank allocation question that this project studies. To my knowledge,
no previously published LoRA variant uses an explicit per-rank
parameter- or activation-cost denominator in its importance score.

\section{Method}
\label{sec:method}

\subsection{Allocator rule}
\label{sec:method-rule}

Let $\theta_i = (A_i, B_i)$ denote the LoRA parameters at target module
$i$, with $A_i \in \mathbb{R}^{r \times d_i^{\text{in}}}$ and
$B_i \in \mathbb{R}^{d_i^{\text{out}} \times r}$. During the warmup
phase, an exponential moving average of the Frobenius gradient norm is
maintained for each module:
\begin{equation}
g_i^{(t)} \;=\; \beta\, g_i^{(t-1)} \;+\; (1-\beta)
   \bigl(\|\nabla_{A_i} \mathcal{L}\|_F + \|\nabla_{B_i} \mathcal{L}\|_F\bigr),
\end{equation}
where the average is seeded at the first observation rather than at
zero in order to avoid an initial bias toward small values. The
per-rank parameter cost
\begin{equation}
c_i \;=\; d_i^{\text{in}} + d_i^{\text{out}}
\end{equation}
gives the exact number of trainable parameters added by a single unit
of rank at module $i$. The combined score $s_i = g_i / c_i^{\alpha}$
is then mapped to integer ranks by a budget-preserving allocator that
first floors each $r_i$ at $r_{\min}$, then distributes the remaining
budget across modules in proportion to $s_i$, caps each $r_i$ at
$r_{\max}$, and finally rebalances by $\pm 1$ deterministically until
$\sum_i r_i = B$ exactly.

\subsection{Two-stage training}
\label{sec:method-twostage}

Stage 1 trains a uniform-rank LoRA model for $W$ warmup steps and
accumulates $g_i$ through the gradient hook described above. At the
boundary between warmup and stage 2, the allocator is run once inside
an instrumented timing block whose wall-clock is recorded as a
separate scheduler-overhead metric. The warmup LoRA weights are then
discarded entirely: stage 2 rebuilds the LoRA model from a fresh copy
of the base, this time with non-uniform per-module ranks taken from
the allocator's output, and trains to completion using the same
hyperparameters as stage 1. The reason for keeping warmup short is
that its purpose is to surface a usable gradient signal for the
allocator, not to pre-train the adapter weights themselves.

A dynamic in-training variant of this scheme, in which the allocator
is re-run periodically rather than only once, is out of scope for this
project. Resizing rank mid-training would require reshaping the
optimizer state of every adapter, which adds a layer of implementation
complexity that runs against the simplicity goal stated in the
introduction. A dynamic version is therefore left as future work.

\begin{algorithm}[t]
\caption{Hardware-Aware Two-Stage LoRA}
\label{alg:hwlora}
\begin{algorithmic}
\STATE \textbf{Input:} base model, dataloader, total budget $B$,
       $r_{\min}$, $r_{\max}$, $\alpha$, EMA decay $\beta$,
       warmup steps $W$, total steps $T$
\STATE Build LoRA model with uniform rank $B/n$ over $n$ targets
\STATE Initialize $g_i \leftarrow \perp$ for each target
\FOR{$t = 1$ to $W$}
   \STATE Forward, backward, update $g_i$ via Frobenius EMA
   \STATE Optimizer step
\ENDFOR
\STATE Begin scheduler-overhead timing block:
\STATE \quad Compute $s_i = g_i / c_i^{\alpha}$
\STATE \quad Allocate $\{r_i\}$ s.t.\ $\sum_i r_i = B$,
       $r_{\min} \le r_i \le r_{\max}$
\STATE End timing block
\STATE Reload base model; build LoRA with non-uniform ranks $\{r_i\}$
\FOR{$t = W{+}1$ to $T$}
   \STATE Forward, backward, optimizer step
\ENDFOR
\end{algorithmic}
\end{algorithm}

\section{Experimental Setup}
\label{sec:setup}

\textbf{Task.} SST-2 binary sentiment classification (67k training,
872 validation) from GLUE \cite{wang2018glue, socher2013sst}.

\textbf{Model.} \texttt{distilbert-base-uncased}
\cite{sanh2019distilbert}.

\textbf{LoRA targets.} The four projection types \texttt{q\_lin},
\texttt{v\_lin}, \texttt{lin1}, and \texttt{lin2} are adapted, giving
24 modules in total: 12 attention projections at a parameter cost of
$c_i = 768 + 768 = 1536$ each, and 12 FFN projections at
$c_i = 768 + 3072 = 3840$ each. The $2.5\times$ heterogeneity in $c_i$
between attention and FFN modules is an important property of this
setup, since on attention-only target sets every $c_i$ would be
identical, and $\alpha = 0$ and $\alpha = 1$ would produce identical
allocations, making the gradient-only ablation degenerate.

\textbf{Rank budget.} A fixed total rank budget of $B = 192$ is used
across all methods, corresponding to 24 modules at a uniform rank of
8. This budget is held invariant across the four compared methods, so
that any observed difference in performance cannot simply be
attributed to one method using more trainable parameters than another.

\textbf{Methods.} Four methods are compared in this evaluation:
Uniform LoRA, AdaLoRA, Gradient-Adaptive LoRA ($\alpha = 0$), and
Hardware-Aware LoRA ($\alpha = 1$). The $\alpha$-sweep ablation
additionally includes $\alpha = 0.5$.

\textbf{Shared training settings.} All methods share the same
optimizer (AdamW), learning rate ($2 \times 10^{-4}$), batch size
(32), maximum number of training steps (3 epochs over SST-2), and the
same training loop, enforced through a single dispatcher. AdaLoRA's
per-step \texttt{update\_and\_allocate} hook is timed using the same
scheduler-overhead instrumentation as the two-stage allocator, so
that its allocation cost is captured under the same metric.

\textbf{Seeds.} Each configuration is run with three random seeds:
42, 43, and 44.

\textbf{Hardware.} All experiments are run on a single NVIDIA RTX
4070 Laptop GPU with CUDA 12.4 and PyTorch 2.6.0+cu124. The full
sweep finished in $15{,}780$ seconds (approximately $4.4$ hours) over
$5 \times 3 = 15$ runs, all of which completed successfully.

parameter counts alone would suggest (Fig.~\ref{fig:peakmem}).
Uniform LoRA sits at $949$ MB and AdaLoRA at $964$ MB, while
Hardware-Aware reaches $1208$ MB and Gradient-Adaptive $1211$ MB.

memory compared to Uniform, despite the fact that Hardware-Aware
actually uses fewer parameters. This is because
\texttt{torch.cuda.max\_memory\_allocated} reflects activation and

footprint of the corresponding $A$ and $B$ factors along the forward
and backward pass also grows. The current cost function $c_i =
d_i^{\text{in}} + d_i^{\text{out}}$ only captures parameter cost per
rank, and does not model activation memory; this is a direction
worth investigating in future work.

\subsection{Asymmetric skew across attention projections}
\label{sec:results-skew}

A closer look at the rank allocation heatmap (Fig.~\ref{fig:heatmap})
shows that the attention-share growth observed above is not symmetric
across the two attention projections. As $\alpha$ grows, rank tends to
migrate from \texttt{ffn.lin1} into \texttt{attn.v\_lin}, while
\texttt{attn.q\_lin} stays low-rank across all methods, since its
gradient signal is too small for the cost penalty alone to lift it.
\texttt{ffn.lin2} stays low across all methods for similar reasons.
As a result, most of the $39\%$-to-$56\%$ growth in attention rank
share reported in Table~\ref{tab:alpha} is driven primarily by
\texttt{v\_lin}, not by both attention projections evenly.

This points to a structural property of the rule
$s_i = g_i / c_i^{\alpha}$: the cost term reshapes allocations only
\emph{among} modules that already carry non-trivial gradient signal.
It cannot create importance for modules whose $g_i$ is near zero,
since the penalty is multiplicative rather than additive. Three
implications follow.

\textbf{1. ``Hardware-aware'' is more precisely ``hardware-biased
gradient'' allocation.} The cost term shapes the distribution of rank
among modules that already have a usable learning signal, but it does
not pull in additional modules from outside that set. This is a
sharper claim than the more general ``the cost term reshapes
allocations'', and it is the one that the data here actually supports.

\textbf{2. This likely explains the small accuracy gap between the two
two-stage variants.} Hardware-Aware ($\alpha = 1$) and
Gradient-Adaptive ($\alpha = 0$) end up in the same accuracy range
($0.904$--$0.906$) because they allocate among the same set of
gradient-signal-rich modules, and differ only in the proportions
assigned to those modules. If the cost penalty were able to reach
\texttt{attn.q\_lin}, then a larger accuracy gap in either direction
might be expected. Its absence is consistent with the picture of
redistribution within a fixed shortlist of modules, rather than
expansion of that shortlist.

\textbf{3. This points to an additive variant as a natural follow-up.}
A rule of the form $s_i = g_i + \lambda / c_i$ would allow the cost
term to contribute independently of $g_i$, and could in principle
route rank toward low-gradient modules. Whether that is actually
desirable is a separate question; funding modules that are not
learning at all may simply waste budget. Even so, comparing the two
rules would help disambiguate whether \texttt{attn.q\_lin}'s low rank
reflects genuine uninformativeness, or just insufficient warmup
signal in the present setup.

\section{Limitations}
\label{sec:limitations}

\begin{itemize}
\item \textbf{Single task and single model.} The evaluation in this
      paper is restricted to SST-2 with DistilBERT-base. Whether the
      allocator's behavior generalizes to multi-class GLUE tasks, or
      to larger transformer backbones such as RoBERTa or Llama, is
      not validated here.
\item \textbf{Two-stage one-shot reallocation only.} A dynamic
      in-training variant of the allocator is left out of scope, so
      it remains unclear whether running the allocator continuously
      throughout training, instead of only once, could close the
      time-to-target gap against Uniform LoRA on this task.
\item \textbf{Three seeds.} Three random seeds is too few to support
      paired significance testing in any rigorous sense. Only the
      mean and standard deviation across seeds are reported, and the
      reader is left to judge whether the observed differences
      between methods are meaningful.
\item \textbf{Mixed practical-efficiency picture against Uniform LoRA.}
      The Hardware-Aware variant uses fewer trainable parameters and
      achieves the lowest seed variance among the four methods
      considered, but Uniform LoRA still reaches the $0.90$ accuracy
      target faster and uses less peak memory on this task. The
      clearer systems-level wins (throughput, scheduler overhead) are
      observed against AdaLoRA, rather than against Uniform.
\item \textbf{Peak memory not modeled in the cost function.} The
      current cost function $c_i = d_i^{\text{in}} + d_i^{\text{out}}$
      only captures parameter cost per rank, and does not model
      activation memory. The unexpected peak-memory excess of the
      two-stage methods over Uniform LoRA is therefore an open
      follow-up rather than a fully explained outcome.
\item \textbf{Multiplicative penalty cannot rescue near-zero gradient
      signal.} As discussed in \S\ref{sec:results-skew},
      \texttt{attn.q\_lin} stays low-rank across every value of
      $\alpha$ tested because its $g_i$ is small. No setting of
      $\alpha$ in the multiplicative rule
      $s_i = g_i / c_i^{\alpha}$ can route rank to a module that has
      essentially no learning signal at all, and an additive variant
      would likely behave differently in this regard.
\end{itemize}

\section{Conclusion}
\label{sec:conclusion}

The cost term in the allocation rule $s_i = g_i / c_i^{\alpha}$
redistributes rank in a meaningful way among modules that already
carry non-trivial gradient signal. The monotonicity observed in the
$\alpha$-sweep confirms that this redistribution is driven by the
cost term itself, since the gradient signal is held essentially
fixed across the three configurations. Under a fixed $192$-rank
budget on SST-2 with DistilBERT, the proposed allocator achieves the
lowest trainable-parameter count of any method evaluated, comparable
validation accuracy with the lowest seed-to-seed variance, and
roughly a $19\times$ reduction in scheduler overhead compared with
AdaLoRA. At the same time, the rule does not recruit rank for
modules with near-zero gradient signal, since the penalty is
multiplicative rather than additive, and it does not deliver an
across-the-board practical-efficiency improvement over Uniform LoRA
on this task; peak memory and time-to-target both still favor
Uniform LoRA in the present setup.

This project asks whether hardware-aware rank allocation can improve
practical efficiency compared with AdaLoRA under constrained training
budgets. The evidence collected in this study supports a sharpened
version of that claim: when the gradient signal is held fixed, the
cost term shifts \emph{where} rank is allocated among modules that
already contribute to learning, at essentially zero accuracy cost on
SST-2 with DistilBERT. Whether this kind of reshaping also
translates into practical efficiency wins at a larger scale, on
tasks with richer gradient signal across a wider set of modules, or
under additive variants of the rule, remains an open question and a
clear direction for future investigation.
